Overview

Smart Hospital Management System (SHMS) is designed to help small-to-medium clinics manage patients, doctors, appointments, pharmacy stock, diagnostic reports and billing — all in a terminal-based interface. The system is role-aware (Admin, Receptionist, Doctor, Patient, Guest).

Explore Features How to Run OOP Concepts

Features

  • Role-based menus: Admin, Receptionist, Doctor, Patient — each with tailored capabilities.
  • Appointments: Schedule, list, cancel; conflict detection prevents double bookings.
  • Billing: Create itemized bills; optional insurance coverage and total calculation.
  • Pharmacy: Add medicines with quantity and expiry; list and low-stock checks.
  • Diagnostics & Surgery: Add diagnostic reports per patient; schedule surgery actions.
  • Persistence: All major entities persist to CSV-like files for simple storage.

Installation & Usage

Compile & run the single-file console app (example with g++):

$ g++ main.cpp -o SmartHospital
$ ./SmartHospital

Files used at runtime (created/updated in the working directory):
